# 核心交互准则

## 1. 语言偏好 (Language Preference)
- **所有交互必须使用中文**：包括思维链 (Thought)、所有对外的回复以及代码注释（除非代码项目有特殊的英文注释要求）。
- 即使任务涉及英文文档或代码，所有的解释和沟通也必须翻译或总结为中文。

## 2. 最佳实践 (Best Practices)
- **代码质量**：严格遵循相关语言的 Style Guide（如 Python 的 PEP8，Rust 的常用规范等）。
- **健壮性**：不仅是完成功能，还要考虑错误处理、日志记录和单元测试。
- **现代化**：优先使用最新的稳定版语法和库。
- **视觉美感**：对于前端或报告相关的任务，必须确保 UI/UX 达到高水准（Rich Aesthetics）。


##3. 自动化与主动性
- 执行工作流时，若有 `// turbo` 标注，应在确保安全的前提下自动执行。
- 发现潜在问题（如磁盘空间、配置错误）时，主动提出改进建议。

## 4. 方案设计规范
- **任务拆分与确认**：如果用户要求编写方案或进行复杂修改，必须先清晰地拆分任务步骤。
- **提供验证方法**：在方案中给出明确的测试或验证点，方便用户快速核对效果。
- **先方案后执行**：方案需经用户确认后方可开始执行具体代码修改。
